## 🛠️ Step 1：Meta for Developers 設定

### 1-1 建立 App

1. 前往 https://developers.facebook.com/
2. 點「我的應用程式」→「建立應用程式」
3. 類型選「**消費者**」
4. 填入應用程式名稱（例：日央水原發文系統）

### 1-2 啟用 Facebook Login

1. 在 App 後台左側選「新增產品」→「Facebook 登入」→「設定」
2. 在「有效的 OAuth 重新導向 URI」填入：
   ```
   http://localhost:3000/api/auth/callback/facebook
   ```
   （正式上線後加入 Vercel 網址）

### 1-3 取得 App ID 和 App Secret

在「設定」→「基本」找到：
- **應用程式編號**（App ID）
- **應用程式密鑰**（App Secret）→ 按「顯示」後複製

### 1-4 申請必要權限（App Review 前先用測試帳號）

在「應用程式審查」→「權限與功能」申請：
- `pages_show_list`
- `pages_read_engagement`
- `pages_manage_posts` ← **需 App Review 才能給一般用戶**

> ⚠️ **測試期間**：只有 App 管理員和在「角色」頁面新增的測試帳號可以使用完整功能

## 🔑 Step 2：設定環境變數

```bash
# 複製環境變數範本
cp .env.local.example .env.local

# 用文字編輯器打開 .env.local 填入：
FACEBOOK_APP_ID=你的_App_ID
FACEBOOK_APP_SECRET=你的_App_Secret
NEXTAUTH_SECRET=（用以下指令生成）
NEXTAUTH_URL=http://localhost:3000
```

生成 NEXTAUTH_SECRET：
```bash
openssl rand -base64 32
```

## 📦 Step 3：安裝與啟動

```bash

# 啟動開發伺服器
npm run dev
```

打開瀏覽器：http://localhost:3000

## ⚠️ 常見問題

**Q：登入後顯示「應用程式未完成審查」**
A：在 Meta App 後台，把自己的帳號加入「測試人員」角色即可使用

**Q：取得粉專失敗，提示缺少 pages_show_list 權限**
A：需要在 App Review 申請此權限，或在開發模式下用 App 管理員帳號測試

**Q：Node.js 版本需求**
A：建議 Node.js 18.17.0 以上（Next.js 14 需求）
